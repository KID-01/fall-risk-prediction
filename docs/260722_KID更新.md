# 2026-07-22 KID 前端更新日志

> 更新人：KID-01  
> 更新范围：`frontend/src/App.jsx`、`frontend/src/index.css`

本次更新对前端看板进行了全面视觉重设计：统一色彩体系、优化页面布局与交互细节、新增白天/黑夜模式切换功能、改进了风险卡片渐变条效果和仪表盘刻度可读性。所有改动均不涉及新增依赖，仅修改两个已有文件。

## 一、整体视觉重设计

### 1.1 色彩体系升级

采用 Tailwind CSS 色系，替换原有较生硬的配色：

| 色彩角色 | 旧值 | 新值 |
|---------|------|------|
| 绿色（低风险） | `#52c41a` | `#22c55e` |
| 黄色（关注级） | `#faad14` | `#eab308` |
| 橙色（预警级） | `#fa8c16` | `#f97316` |
| 红色（高危级） | `#f5222d` | `#ef4444` |
| 蓝色（主按钮） | `#1890ff` | `#3b82f6` |
| 背景色 | `#f0f2f5` | `#f1f5f9` |
| 文字色 | `#1a1e2b` | `#1e293b` |

### 1.2 新增 CSS 变量体系

```css
--text / --text-secondary / --text-muted     /* 三层文字色阶 */
--bg / --card-bg / --border                  /* 背景与边框 */
--shadow-sm / --shadow / --shadow-lg         /* 三层阴影 */
--radius / --radius-sm                        /* 统一圆角 */
--transition                                   /* 统一过渡动画 */
--alert-item-bg / --alert-item-hover          /* 告警列表背景 */
--chart-bg / --chart-text / --chart-split / --chart-axis  /* 图表配色 */
```

### 1.3 字体优化

- 新增中文字体优先级：`PingFang SC` → `Microsoft YaHei` → 系统兜底
- 开启 `-webkit-font-smoothing: antialiased` 字体平滑

## 二、页面布局改进

### 2.1 标题栏

- 新增副标题："基于多模态 AI 监测 · 家属端实时看板"
- 连接状态改为圆角胶囊标签（`.connection-badge`）
- 在线时绿色圆点带发光效果（`box-shadow`）

### 2.2 控制按钮

| 按钮 | 改动 |
|------|------|
| 启动监控 | 添加 ▶ 图标，蓝色发光阴影 |
| 停止监控 | 添加 ■ 图标，红色发光阴影 |
| 重置基线 | 添加 ↻ 图标，改为白底灰色边框样式 |
| 通用 | 点击时 `scale(0.97)` 微缩动画 |

### 2.3 风险等级卡片

- 底部指标改为三列卡片式布局：「基线采集」「处理帧数」「有效帧数」
- 数字 22px 加粗，标签 13px 灰色
- 基线完成时显示 ✓ 而非数字

### 2.4 图表区域

- 图表卡片悬停时阴影加深
- 标题前添加蓝色/红色竖条装饰
- 趋势图空状态：显示 "📊 暂无历史数据，启动监控后数据将在此展示"
- 仪表盘指针改为蓝色，增加动画过渡

### 2.5 告警列表

- 告警标签改为圆角胶囊样式（`border-radius: 20px`）
- 每条告警独立背景色，悬停变色
- 空状态显示 "🔔 暂无告警记录"
- 时间格式改为 `toLocaleString('zh-CN')`

### 2.6 动画效果

- 卡片加载时 `fadeInUp` 淡入上浮动画（依次延迟 0s / 0.1s / 0.2s）
- 高危状态时 `pulse-border` 边框脉冲动画（2 秒周期）

## 三、白天/黑夜模式切换

### 3.1 功能

- 右上角新增圆形主题切换按钮：🌙（亮色模式）/ ☀️（暗色模式）
- 选择自动保存到 `localStorage`，刷新不丢失
- 首次打开时自动跟随系统偏好（`prefers-color-scheme`）

### 3.2 暗色模式适配

| 元素 | 亮色模式 | 暗色模式 |
|------|---------|---------|
| 背景 | `#f1f5f9` | `#0f172a` |
| 卡片 | `#ffffff` | `#1e293b` |
| 文字 | `#1e293b` | `#e2e8f0` |
| 次要文字 | `#64748b` | `#94a3b8` |
| 边框 | `#e2e8f0` | `#334155` |
| 告警条目背景 | `#f8fafc` | `#1e293b` |
| 阴影 | 浅色半透明 | 深色半透明 |

### 3.3 ECharts 图表暗色适配

- 仪表盘：刻度数字暗色下用 `#cbd5e1`（亮灰），亮色下用 `#64748b`（深灰）
- 趋势图：坐标轴、分割线、提示框背景全部跟随主题
- 图表在 `theme` 状态变化时自动重绘

## 四、风险卡片渐变条优化

### 4.1 从实线到渐变

- 之前：4px 纯色实线
- 之后：48px 渐变条，从顶部向下方自然淡化至透明

### 4.2 亮色模式

- 渐变透明度：顶部 0.28 → 中部 0.10 → 底部透明
- 每种风险等级对应独立颜色渐变（绿/黄/橙/红）

### 4.3 暗色模式

- 卡片背景从深色固有色过渡到 `#1e293b`（卡片底色），40% 处融合
- 渐变条透明度：顶部 0.10 → 中部 0.03 → 底部透明（暗色下不宜过亮）
- 绿底 `#0a2e1a`、黄底 `#3d2206`、橙底 `#3d1a08`、红底 `#3d0e0e`

## 六、技术细节

### 6.1 暗色模式实现方式

使用 `document.documentElement.setAttribute('data-theme', theme)` 在 `<html>` 上设置属性，CSS 通过 `[data-theme="dark"]` 选择器覆盖变量。

### 6.2 错误处理改进

所有 `fetch` 请求的 `catch` 块改为静默处理（移除 `console.error`），避免后端未启动时控制台刷屏。

### 6.3 依赖变化

无新增依赖，仅修改已有文件。
