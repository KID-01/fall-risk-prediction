## 模块 2：数据采集管线

**时间：6月6日 — 6月20日（2周）**

### T2.1 视频采集脚本 [✅]
- **优先级：** P0 | **预估：** 1天
- `scripts/collect_video.py` 参数化采集，自动生成元数据
- 支持批量采集计划（CSV驱动）

### T2.2 数据预处理管线 [✅]
- **优先级：** P0 | **预估：** 2天
- 帧提取 → YOLOv8-nano 人体检测 → ROI裁剪（256×256）
- `torchvision.transforms` 构建可配置 Pipeline

### T2.3 骨骼关键点提取管线 [✅]
- **优先级：** P0 | **预估：** 2天
- **方案：使用 YOLOv8-Pose**，一次推理同时完成人体检测+17点COCO关键点提取，无需独立人体检测器
- **统一关键点抽象层**：定义13个语义枚举（PoseKeypoint），通过映射表支持 YOLO-Pose(17点) 和 MediaPipe(33点) 双后端
- `convert_keypoints()` 将各后端原始坐标统一转换为 (13,4) 格式，特征计算代码后端无关
- 批量提取脚本 `scripts/extract_keypoints.py`，存储为 `.npy` shape=(T,13,4)

### T2.4 标注工具配置与数据管理
- **优先级：** P0 | **预估：** 1.5天
- LabelStudio Docker 部署，定义标注Schema
- 标注一致性检查（Cohen's Kappa）

### T2.5 PyTorch Dataset与DataLoader [✅]
- **优先级：** P0 | **预估：** 2天
- `FallRiskDataset`：时序采样（T=90帧），数据增强
- 按被试者分层划分：7:1.5:1.5

## 模块 3：姿态估计与步态特征提取

**时间：6月21日 — 7月10日（3周）**

### T3.1 姿态估计推理引擎 [◐]
- **优先级：** P0 | **预估：** 1.5天
- **方案：YOLOv8-Pose 封装为 `YoloPoseExtractor`**，一次推理同时输出人体框+17点关键点
- `create_keypoint_extractor(backend)` 工厂函数，配置 `pose_estimation.backend` 一行切换 yolo_pose / mediapipe
- 统一输出 `KeypointFrame(13,4)`，YOLO-Pose 像素坐标自动归一化并映射为13语义关键点
- 相比"独立检测+MediaPipe"两次推理，减少约40%延迟

### T3.2 步态特征提取器 [◐]
- **优先级：** P0 | **预估：** 3天
- `GaitFeatureExtractor` 提取 ~80-120维特征
- 核心接口：
  ```python
  class GaitFeatureExtractor:
      def __call__(self, keypoints: np.ndarray) -> np.ndarray:
          # keypoints: (T, 33, 3) -> features: (D,)
  ```

### T3.3 步态特征有效性验证 [◐]
- **优先级：** P1 | **预估：** 1天
- t-SNE可视化、SHAP特征重要性、Mann-Whitney U检验

## 技术风险与应对

| 风险 | 应对策略 |
|------|----------|
| **时间紧张** | 并行推进多个模块；优先保证核心功能（P0任务）；采用敏捷迭代策略 |
| **RAM/显存不足** | INT8量化；减小batch size；梯度检查点 |
| **萤石SDK兼容性** | 锁定SDK版本；Docker固定环境 |
| **姿态估计精度不足** | 卡尔曼滤波平滑；数据增强；单独报告困难场景 |
| **模型过拟合** | 公开数据集预训练；强数据增强；Dropout+早停 |
| **实时性不达标** | ONNX Runtime加速；降低输入帧率；模型剪枝 |
